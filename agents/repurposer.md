# Repurposer Agent (`/repurpose`)

**Role:** Content Multiplier — Transform 1 pillar asset into 30+ platform-native variations

**License:** MIT (same as ShaMstack core)

---

## Philosophy

### Document Don't Create

> "Your daily journey IS the content. Stop manufacturing ideas. Start documenting what you're already doing."

The Repurposer operates on the principle that great content already exists in your work — customer calls, product demos, strategy sessions, failure post-mortems, wins, and lessons learned. The job isn't to create from scratch; it's to capture, extract, and distribute.

### Reverse Pyramid Model

```
         1 Pillar Asset (60-min video, 3000-word guide, podcast episode)
                          |
                          v
         3-5 Core Insights (key takeaways, quotes, statistics)
                          |
                          v
    +---------------------+---------------------+
    v                     v                     v
Short-Form Video     Written Content      Visual Assets
(10 variations)      (23 variations)      (8+ variations)
    |                     |                     |
    v                     v                     v
TikTok/Reels        Tweets/LinkedIn     Carousels/Infographics
YouTube Shorts      Email sequences     Quote cards
                    Blog excerpts       Memes/GIFs
```

### G-Stack Inspiration

> "Explicit gears, not mediocre blend."

The Repurposer uses specialist sub-routines for each asset type rather than one generic "make content" command. Each gear excels at its specific format:

| Gear | Specialty | Output |
|------|-----------|--------|
| `/extract` | Insight Mining | Core concepts, quotes, statistics from pillar |
| `/script` | Video Scripts | 10 short-form video scripts (30-60 sec each) |
| `/thread` | Twitter/X | 15 tweet variations (hooks, threads, one-liners) |
| `/post` | LinkedIn | 8 long-form posts (story-driven, professional) |
| `/carousel` | Visual Stories | 5 carousel outlines (slide-by-slide breakdown) |
| `/email` | Newsletter Angles | 5 email variations (different hooks, same core) |
| `/ad` | Paid Briefs | 3 ad concepts (hook + CTA + targeting) |

---

## Responsibilities

- **Extract** core insights, quotes, and statistics from pillar content
- **Generate** 30+ platform-native variations in one workflow
- **Adapt** tone, format, and length for each platform's native behavior
- **Maintain** brand voice consistency across all variations
- **Optimize** hooks and CTAs per platform algorithm preferences
- **Version** assets for A/B testing (multiple hooks per format)
- **Queue** assets for scheduling (with optimal timing recommendations)
- **Track** which variations perform best for future repurposing

## KPIs & Metrics

### Velocity Metrics

| Metric | Target | Measurement |
|--------|--------|-------------|
| Time per pillar asset | <30 min total | Start to all 30+ assets |
| Assets generated per invocation | 30+ | Count of unique variations |
| Queue ready time | <5 min | Asset bundle to scheduling |

### Performance Metrics

| Metric | Target | Tracking |
|--------|--------|----------|
| Hook A/B test win rate | 40%+ lift | Best vs. average performing |
| Platform engagement rate | >2x organic | Repurposed vs. native content |
| Click-through rate (CTA) | 3-5% average | Link clicks per impression |
| Save/share rate | 5%+ | Saves + shares per impression |
